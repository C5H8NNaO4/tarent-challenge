post /trainings/{id}/slot/
Add a timeslot to a training. (addTimeSlotToTraining)

Path parameters

id (required)
Path Parameter — The training id

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body string (optional)
Body Parameter — The updated/added timeslot of the training.

Query parameters

Return type

Booking

Example data

Content-Type: application/json
{
  "trainingId" : 0,
  "timeSlot" : "25.07 15:30",
  "userId" : 6
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

successful operation Booking

403

unauthorized Failure
